# Roadmap

EvalAI evolves through community contributions and CloudCV maintenance. There is no fixed public date-based roadmap; priorities follow hosted platform needs and open [GitHub issues](https://github.com/Cloud-CV/EvalAI/issues).

## How to influence direction

- Open feature requests with clear use cases (challenge hosting, participants, or developers).
- Contribute documentation fixes via pull requests under `docs/source/`.
- Join [community Slack](../07-troubleshooting/support/community-support.md).

## Documentation improvements

Recent work reorganized docs under numbered sections (`01-getting-started/` through `08-reference/`). Legacy URLs redirect to the new structure — see `.readthedocs.yaml`.

## Architecture notes

Historical design decisions may be recorded in [architecture_decisions.md](../architecture_decisions.md) in the source tree.
